Taskbar Music Lounge AE — Changelog
=====================================

v1.0.0 — 2026-05-08
---------------------
Initial release of the Audiobook Edition fork.
Based on Taskbar Music Lounge v4.0.1 by Hashah2311 (MIT license).

  Upstream bug fixes (resolved before v1.0.0 release)
  - Null dereference on window creation failure.
  - WM_APP message collision (APP_WM_CLOSE = WM_APP).
  - Dead g_Running flag removed.
  - SendMediaCommand targeted wrong GSMTC session.
  - Blocking WinRT IO ran on the message thread.
  - IDT_ANIMATION timer not killed when scrolling stopped.
  - LOWORD/HIWORD mouse coords broke on multi-monitor setups.

  Libby (libbyapp.com) audiobook support
  - Strips "Libby - Open: " prefix so the widget shows just the book title.
  - Fetches cover art from Open Library Covers API; two-pass search (title
    field then full-text), subtitle stripping, exact title matching only —
    never shows a cover for the wrong book.
  - 5-entry in-memory LRU cache; failed titles stored as sentinels so they
    are not re-fetched within the same session.
  - Right-click cover menu (Libby only): Try Different Cover, Remove Cover,
    Lock/Unlock Cover (pins image, greys destructive items), Restore Cover.

  Additional fixes
  - Cover fetch runs outside g_MediaState.lock (no lock held during HTTP IO).
  - Race condition: Try Different Cover no longer reverts within one second;
    poll timer inherits exclusion list and cache entries are updated in-place.
